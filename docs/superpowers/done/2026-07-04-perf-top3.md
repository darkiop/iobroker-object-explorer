# Top-3 Perf Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the three highest-impact performance issues from `docs/performance-analysis.md`: unmemoized `PanelContextValue` causing StateList re-renders on every filter keystroke, a broad `invalidateQueries` in `useCreateDatapoint`, and missing code-splitting for rarely-used modals.

**Architecture:** Task 1 wraps `panel1Value` in `useMemo` with a precise dependency array so `PanelContext.Provider` only produces a new object identity when a field StateList actually reads changes. Task 2 replaces `useCreateDatapoint`'s `invalidateQueries` with a targeted `setQueriesData` insert, matching the pattern already used by `useExtendObject`/`useDeleteObject`/`useRenameDatapoint`. Task 3 converts the remaining eagerly-imported modals in `App.tsx` to `React.lazy` + `Suspense`, mirroring the existing `HistoryModal` pattern.

**Tech Stack:** React 18, TanStack Query v5, Vite, Vitest + @testing-library/react

- [ ] **Step 4: Replace `onSettled` invalidate with a targeted patch**

In `src/hooks/useObjectMutations.ts`, replace lines 95-127 (`useCreateDatapoint`) with:

```ts
export function useCreateDatapoint() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: async ({ id, common, initialValue, objectType = 'state', roomEnumId, functionEnumId }: {
      id: string;
      common: Partial<IoBrokerObjectCommon>;
      initialValue?: string;
      objectType?: 'state' | 'folder' | 'device' | 'channel';
      roomEnumId?: string | null;
      functionEnumId?: string | null;
    }) => {
      await createObject(id, common, objectType);
      if (objectType === 'state' && initialValue !== undefined && initialValue !== '') {
        await setState(id, initialValue);
      }
      if (roomEnumId) {
        await updateRoomMembership(id, null, roomEnumId);
      }
      if (functionEnumId) {
        await updateFunctionMembership(id, null, functionEnumId);
      }
    },
    onSuccess: (_data, { id, common, objectType = 'state' }) => {
      queryClient.setQueriesData(
        { queryKey: queryKeys.objects.root },
        (old: Record<string, IoBrokerObject> | undefined) => {
          if (!old) return old;
          return {
            ...old,
            [id]: {
              _id: id,
              type: objectType,
              common: common as IoBrokerObjectCommon,
              native: {},
            } as IoBrokerObject,
          };
        }
      );
    },
    onSettled: (_data, _error, vars) => {
      if (vars.roomEnumId) {
        queryClient.invalidateQueries({ queryKey: queryKeys.metadata.roomMap });
      }
      if (vars.functionEnumId) {
        queryClient.invalidateQueries({ queryKey: queryKeys.metadata.functionMap });
      }
    },
  });
}
```

Note: the patched object is a best-effort shape (matches what the tree/table need to render immediately); the object gets fully corrected on next natural refetch since `staleTime` eventually elapses or a manual refresh occurs. This mirrors what `useExtendObject`/`useDeleteObject` already do for partial local objects.
